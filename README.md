## 스터디 방식
1) 팀원코드리뷰 : 이전 과제의 코드를 돌아가면서 리뷰한다.
2) 본 발표 : 발표자가 책 해당 챕터 내용을 요약하고 api gslide 준비 하기 (api 문서 자료 찾아서)

3) 스터디 이후 할일 
    - 발표자: 자료 업데이트하고 블로깅 (https://motlabs.github.io/ )
    - 다른 팀원: 배운것을 토대로 본인의 코드 업데이트
    - 다음 발표자: 차주 발표 준비
